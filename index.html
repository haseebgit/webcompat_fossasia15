UA detection is about determining a browser or device's capabilities by recognizing the user agent string that is sent with each request. UA detection is brittle.

UA detection uses

  • Capability identification > Use feature detection
  • Serve mobile/tablet/desktop content > use media queries

Avoid UA detection!

There are alternatives to UA detection.

Feature Detection

                var v = document.createElement("video");
                if(v.canPlayType) { 
                   var webm = v.canPlayType('video/webm');
                   var h264 = v.canPlayType('video/mp4;
 codecs="avc1.42E01E, mp4a.40.2"');
                }
Use feature detection to have the device tell you about its capabilities.

Media Queries

                <link href="site.css" type="text/css" media="screen
 and (max-device-width: 480px)" rel="stylesheet"/>
            
                @media screen and (max-width:480px) {
                   …
                }
Create a responsive site that dynamically adapts to device screen size with media queries.

If you must use UA detection stay as broad as possible

  • Detect Mobi for mobile devices
    ("and not 'iPad'" if you want to avoid serving mobile content to iPads)
  • Detect Tablet or iPad or Touch for tablet based devices
    (see how this gets messy)

    • If you must use UA detection, stick to broadly identifying categories of devices such as mobile and tablet instead of detecting specific devices.
